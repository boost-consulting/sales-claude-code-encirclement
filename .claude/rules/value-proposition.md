# 価値訴求力

「無敗営業」第5章より。お客さまに価値を感じてもらい、
関係を深めていくためのフレームワーク。

---

## トリガー条件

以下のような質問・相談があった場合にこのルールを適用：
- 「お客さんに価値を感じてもらえない」
- 「関係を深めたい」
- 「情報を教えてもらえない」
- 「また会いたいと思ってもらえない」
- 「業者扱いされる」

---

## 核心メッセージ

> **質問力だけでは不十分。お客さまから情報を引き出すには、先に価値を提供する必要がある**

---

## 質問力と価値訴求力の関係

| 概念 | 役割 |
|------|------|
| **質問力** | お客さまから情報をもらう（借り） |
| **価値訴求力** | お客さまに価値を提供する（貸し） |

**重要な原則:**
- 質問力と価値訴求力は**コインの表と裏**
- 「借り」が多いと情報を出し渋られる
- 「貸し」（価値提供）が多いと情報を提供してくださる

---

## 重要データ

> **「営業担当者としての魅力や価値を感じない」と回答した方 = 43.7%**

価値訴求力不足の特徴：
- 回答が**それほど極端な偏りを見せず、バラけている**
- = **価値の感じ方は人それぞれ**

---

## 価値の四象限

```
        情緒価値（感情軸）
              ↑
    ┌─────────┼─────────┐
    │ 好感・共感 │ プラスα・提言 │  高
    │         │           │
    ├─────────┼─────────┤
    │労務提供・  │情報提供・    │  低
    │適量ｺﾐｭﾆｹｰｼｮﾝ│人の紹介     │
    └─────────┴─────────┘
         低          高
              機能価値（思考軸）→
```

### 各象限の詳細

#### 左下：労務提供・適量コミュニケーション（難易度：低）
- 「作業が楽になった」「手伝ってもらって助かった」
- 「ちょうどいいときに連絡をくれる」
- **最初の一歩として最適**

#### 左上：好感・共感（情緒価値が高い）
- 「この人と一緒にいて気持ちいい」
- 「この人は自分のことをわかってくれる」
- ビジネスマナー、人柄・人格

#### 右下：情報提供・人の紹介（機能価値が高い）
- 「それは知っていた方がいい」
- 「人を紹介してもらって助かる」
- 業界知識、製品知識、人脈

#### 右上：プラスα・提言（難易度：高）
- 「それは思いつかなかった！」
- 「あえて言ってもらえるのはありがたい」
- 付加価値のある提案、外部からのアドバイス

---

## 価値訴求のレベルアップの流れ

```
労務提供・適量コミュニケーション
        ↓ 感謝「やってくれてありがとう」
  自分しか持っていない情報の発見
        ↓
    情報提供・人の紹介
        ↓ 感謝「教えてくれてありがとう」
      仮説の着想
        ↓
      プラスα・提言
        ↓ 感謝「言ってくれてありがとう」
```

> **この縦ラインを支えるのが「好感・共感」**

## 5つのCフレームワーク

情報提供や人の紹介のニーズを考えるために押さえるべき5つの観点。

### 基本の3C

| C | 意味 |
|---|------|
| **C**ustomer | お客さま |
| **C**ompetitor | 競合 |
| **C**ompany | 当社 |

### 追加の2C

| C | 意味 | 考えるべき質問 |
|---|------|--------------|
| **C**ustomer's **C**ustomer | お客さまのお客さま | お客さまはどのような戦略でビジネスを展開しているか？ |
| **C**ustomer's **C**ompetitor | お客さまの競合 | お客さまはどこをベンチマークしているか？ |

## ジョハリの窓

お客さまとの関係を深める順番。

```
              │ 自分は知っている │ 自分は気づいていない
─────────────┼─────────────────┼──────────────────
お客さまは    │ ①開放の窓       │ ③盲点の窓
知っている    │ （スタート地点） │ （フィードバックをもらう）
─────────────┼─────────────────┼──────────────────
お客さまは    │ ②秘密の窓       │ ④未知の窓
気づいていない│ （自分から開示） │ （深い気づきが生まれる）
```

**順番:** 開放 → 秘密 → 盲点 → 未知

**ポイント:** 秘密を共有することで関係が深まる

## 実践アドバイス

### 入り口は「労務提供」「適量コミュニケーション」から

**理由:**
- 比較的難易度が低い
- 自分の時間さえ捻出すれば実践できる
- まだ関係が構築できていないお客さまにも有効

**具体例:**
- お客さまの社内資料を肩代わりして作成
- お客さまのドキュメント・テンプレートを使用
- 定例会議の前後にタイミングよく連絡

### お客さまの業務プロセスを理解する

- どの工程に負担がかかるか把握
- 定例会議のスケジュールを聞いておく
- 会議前後が「お役立ち」の絶好のタイミング

### フィードバックを能動的に求める

> **お客さまは不満があっても口に出さない**

質問例：
- 「本日の商談は100点満点で何点ですか？」
- 「改善点があれば教えてください」

---

## よくある失敗パターン

| 失敗 | 問題点 |
|------|--------|
| 労務提供だけで終わる | 「やってくれてありがとう」で終わってしまう |
| 情報提供がマンネリ化 | ネタが尽きる、響かなくなる |
| いきなりプラスα・提言 | 経験値が必要、一朝一夕にはできない |
| 間の悪いタイミングで連絡 | お客さまの業務を理解していない |

---

## 価値訴求力不足の具体的要因（調査結果）

| 順位 | 足りないもの | 割合 |
|------|------------|------|
| 1 | 適切なコミュニケーションの量や頻度 | 約40% |
| 2 | 人のネットワーク | 約35% |
| 3 | 取り扱い製品に関する知識 | 約30% |
| 4 | 役立つ情報を提供しようという姿勢 | 約30% |
| 5 | 関係を構築しようとする姿勢 | 約28% |
